# 银行IT架构管理平台 — 设计规格书

## 一、平台定位

面向全行人员的IT架构管理平台，统一管理技术基础设施（CMDB）及应用架构、技术架构、数据架构，支持多级审批流程和全景可视化报表。

**分期策略**：
- **第一期**：CMDB基础设施 + 三维架构管理（应用/技术/数据）+ 审批流程 + 基础报表
- **第二期**：监控告警 + 变更流程全生命周期管理

## 二、功能模块

| 模块 | 功能描述 |
|------|----------|
| **CMDB管理** | 基础设施配置管理（服务器、网络、存储、云资源），支持手工录入和文件批量导入 |
| **应用架构** | 应用系统列表、服务架构、模块划分、依赖关系 |
| **技术架构** | 技术栈分布（中间件、数据库、语言框架等）、部署架构 |
| **数据架构** | 数据分布、数据流向、数据实体关系 |
| **审批流程** | 新建/变更应用多级审批，支持流程配置 |
| **报表中心** | 统计报表、三全景图（系统/技术栈/数据分布）、合规报告、趋势分析 |
| **权限管理** | 基于角色的细粒度权限控制，对接行内统一身份认证 |
| **二期：监控告警** | 基础设施状态监控、架构变更监控 |
| **二期：变更管理** | 变更申请、审批、执行、验证全流程 |

## 三、角色与权限

| 角色 | 权限说明 |
|------|----------|
| 架构师/技术专家 | 完整编辑和管理架构资产 |
| 开发/运维人员 | 查看架构 + 发起变更申请，少数人可编辑 |
| 项目经理/业务部门 | 查看架构，了解系统归属和依赖 |
| 科技管理层 | 查看总览和报表，不做具体操作 |
| 审计/合规人员 | 只读，查看变更历史和配置记录 |
| 系统管理员 | 平台配置、用户管理、权限配置 |

## 四、数据入口

- **手工录入**：单条录入资产/架构信息
- **文件导入**：支持Excel/CSV批量导入
- **审批新建**：通过审批流程创建新应用及相关信息

## 五、技术架构

- **前端**：Vue.js（前后端分离）
- **后端**：Spring Boot（Java）
- **数据库**：MySQL/PostgreSQL
- **部署**：私有化部署

### 后端服务模块

| 服务 | 职责 |
|------|------|
| CMDB服务 | 基础设施配置管理 |
| 架构服务 | 应用/技术/数据架构管理 |
| 审批服务 | 多级审批流程引擎 |
| 报表服务 | 统计数据生成、全景图渲染 |
| 权限服务 | 角色权限管理、对接行内身份认证 |
| 用户服务 | 用户账号管理 |
| 文件服务 | 文件上传下载、批量导入导出 |

## 六、三全景图设计

| 全景图 | 内容 |
|--------|------|
| **系统全景图** | 全行应用系统分布、归属部门、生命周期状态 |
| **技术栈全景图** | 全行技术栈使用分布、中间件类型、数据库类型统计 |
| **数据分布全景图** | 各系统数据分布、数据流向、数据存储情况 |

## 七、报表体系

- **统计报表**：系统数量统计、技术栈分布、部门分布、生命周期分布
- **全景图**：系统全景图、技术栈全景图、数据分布全景图
- **合规报告**：配置变更记录、权限变更历史、架构变更轨迹、监管合规报告导出
- **趋势分析**：架构演进趋势、系统老化分析

## 八、一期功能范围

**包含**：
- CMDB基础设施管理（服务器、网络、存储、云资源）
- 应用架构管理
- 技术架构管理
- 数据架构管理
- 多级审批流程
- 权限与用户管理
- 报表中心（含三全景图）
- 文件导入导出

**不包含（放入二期）**：
- 监控告警
- 变更全流程管理
- 与行内其他系统的深度集成

## 九、二期功能范围

- 监控告警（基础设施状态监控、架构变更监控）
- 变更管理全流程（申请 → 审批 → 执行 → 验证）
