## 一、测试范围

本次测试涵盖登录认证、菜单权限控制、角色权限配置三大功能模块。

### 1.1 测试对象

| 模块 | 测试内容 |
|------|----------|
| 后端API | 认证接口、菜单管理接口、角色权限接口 |
| 前端页面 | 登录页、首页动态菜单、角色管理权限配置 |
| 整体流程 | 登录->菜单展示->权限控制->登出 |

### 1.2 测试环境准备

执行数据库迁移脚本：
```bash
mysql -u root -p < .worktrees/development/database/migration_v1.3_auth.sql
```

| admin | admin123 | 超级管理员 | 拥有所有菜单权限 |

## 四、风险与注意事项

### 4.1 已知风险

| 风险 | 缓解措施 |
|------|----------|
| admin密码已在数据库中更新为MD5格式 | 确保迁移脚本正确执行 |
| JWT Token存储在localStorage | 考虑未来升级为httpOnly Cookie |
| 前端菜单树首次加载可能需要等待 | 添加loading状态 |

### 4.2 测试注意事项

1. 每次测试前清除浏览器缓存和localStorage
2. 测试时使用无痕模式避免登录状态干扰
3. API测试需要先启动后端服务 (mvn spring-boot:run)
4. 前端测试需要先启动前端服务 (npm run dev)
